Сравнение между 6те имплемнтации на API server-a.

Express

Express е най-лесният framework за начинаещи.
С него бързо може да се създаде работещ сървър.

Предимства

  • Бърза имплемeнтация.
  • Малък обем код.
  • Използва се от много разработчици.

Недостатъци

  • Не е подходящ за големи проекти.
  • Няма организация на файловете.

Actionhero

Actionhero е framework разробетен около концепцията за actions.
Това са отделни логики, който се извикват, когато е нужно.
Например, когато се достъпи определен endpoint.

Предимства

  • Добра организация на файловете.
  • Лесно създаване на routes.
  • Готова имплемeнтации на swagger.

Недостатъци

  • Лоша документация.
  • По-малко популярен е.

Hapi

Hapi е прост framework, който има няколко вградени функции.
Една от тях е plugins. Може сам да си направиш собствен или да използваш някой от готовите.
Просто добавяш един ред код към главния файл и plugin-ът работи.

Предимства

  • Добра организация на файловете.
  • Добра документация.
  • Готови plugin-и.

Недостатъци

  • Няма организация на файловете.

Koa

Koa framework по същество е много близък до Express. Това го прави лесен за използване.
Има почти същия стил на писане с много-малки разлики.

Предимства

  • Малък обем код.
  • Добра документация.

Недостатъци

  • Няма организация на файловете.
  • Не е много популярен.

Feathers

Feathers е frameworks с повече готови функционалности.
Той е подходящ за по-големи проекти.

Предимства

  • Добра организация на файловете.
  • Добра документация.
  • Команди за генериране на код.

Недостатъци

  • По-малко популярен е.

Sails

Sails е frameworks, който е подходящ за големи проекти. Съдържа готови функции.
Командите за генериране на код са много удобни. Недостатик е, че за създаване на един endpoint се генерират 3 файла.
Това значително увеличава обема от код.

Предимства

  • Добра организация на файловете.
  • Добра документация.
  • Команди за генериране на код.

Недостатъци

  • Труден за начинаещи.
  • Голям обем код.

Заключение

Всеки framework си има свойте силни и слаби страни. Не можем да използваме един framework за всичко.
За малки проекти бих използвал Express или Koa, защото са по-малки като обем и нямат ненужни неща.
За големи проекти бих използвал Feathers или Sails, защото имат готови команди за генериране на логика.
Също така създаването на routes е улеснено.
